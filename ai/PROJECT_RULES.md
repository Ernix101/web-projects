# Project Rules

## Development Philosophy
Projects should prioritize clarity, learning, and maintainability.

## Development Process
1. Start with a simple working version
2. Improve gradually
3. Refactor when the design becomes clearer

## Code Style
- Prefer readability over cleverness
- Comment important logic
- Keep functions small

## Debugging
When fixing bugs:
1. Identify the cause
2. Explain the issue
3. Apply the fix
4. Verify the solution
